# Практическое задание 35.1

## Задания

### Описание задачи
Ты уже хорошо освоил работу со списками, словарями и базовыми алгоритмами. Теперь мы переходим к основам функций — ключевому инструменту для структурирования кода. В этом блоке ты научишься объявлять функции с помощью `def`, передавать в них данные через параметры и получать результат с помощью оператора `return`. Это фундамент, без которого невозможно двигаться к более сложным темам, таким как значения по умолчанию или переменное число аргументов.

Твоя задача — создать программу для анализа списка оценок студентов. Программа должна уметь вычислять средний балл по всем оценкам и определять, является ли студент "отличником" (средний балл >= 4.5). Вся логика должна быть реализована через функции.

### Пример входных и выходных данных
**Пример 1:**
*   **Входные данные:** Список оценок `[5, 4, 5, 3, 5]`.
*   **Ожидаемый вывод:**
    ```
    Средний балл: 4.4
    Статус: Не отличник
    ```
    *Пояснение:* Сумма оценок (5+4+5+3+5) = 22. Количество оценок = 5. Среднее = 22 / 5 = 4.4. Поскольку 4.4 < 4.5, статус "Не отличник".

**Пример 2:**
*   **Входные данные:** Список оценок `[5, 5, 5, 5]`.
*   **Ожидаемый вывод:**
    ```
    Средний балл: 5.0
    Статус: Отличник
    ```
    *Пояснение:* Сумма = 20, количество = 4, среднее = 5.0. 5.0 >= 4.5, поэтому статус "Отличник".

**Пример 3:**
*   **Входные данные:** Список оценок `[]` (пустой).
*   **Ожидаемый вывод:**
    ```
    Средний балл: 0.0
    Статус: Не отличник
    ```
    *Пояснение:* Для пустого списка сумма = 0, количество = 0. Чтобы избежать деления на ноль, средний балл следует принять равным 0.0.

### Критерии проверки и ограничения
1.  **Обязательное использование функций:** В решении должны быть как минимум две функции, объявленные через `def`.
2.  **Параметры и аргументы:** Функции должны принимать данные через параметры. Передавай список оценок как аргумент при вызове.
3.  **Возврат значения:** Каждая функция должна возвращать результат с помощью `return`. Не используй `print` внутри функций для передачи результата наружу.
4.  **Позиционные аргументы:** Используй только позиционные аргументы (передачу данных по порядку).
5.  **Обработка пустого списка:** Программа должна корректно обрабатывать случай, когда список оценок пуст, и возвращать средний балл 0.0.
6.  **Запрещенные темы:** Не используй значения по умолчанию для параметров, именованные аргументы, `*args`, `**kwargs` или любые темы, связанные с областью видимости. Сосредоточься только на `def`, параметрах, аргументах, `return` и `None`.

### Решение задачи (псевдокод)
1.  Объяви функцию `calculate_average`, которая принимает один параметр — список чисел (оценок).
    1.  Если переданный список пуст, немедленно верни число 0.0.
    2.  Иначе:
        1.  Вычисли сумму всех элементов списка.
        2.  Подсчитай количество элементов в списке.
        3.  Раздели сумму на количество.
        4.  Верни результат деления.
2.  Объяви функцию `check_excellent`, которая принимает один параметр — средний балл (число).
    1.  Если средний балл больше или равен 4.5, верни строку "Отличник".
    2.  Иначе верни строку "Не отличник".
3.  В основной части программы:
    1.  Создай список оценок (например, `[5, 4, 5, 3, 5]`).
    2.  Вызови функцию `calculate_average`, передав ей созданный список в качестве аргумента. Сохрани возвращенное значение в переменную.
    3.  Вызови функцию `check_excellent`, передав ей вычисленный средний балл в качестве аргумента. Сохрани возвращенное значение в другую переменную.
    4.  Напечатай средний балл и статус, используя сохраненные переменные.

### Рекомендации
-   **Вынести в отдельную функцию:** Логику вычисления среднего балла и логику определения статуса студента необходимо вынести в отдельные функции, как описано в псевдокоде. Это соответствует цели блока — научиться декомпозировать задачу на отдельные, переиспользуемые блоки кода.
-   **PEP8:**
    -   Используй осмысленные имена для функций и переменных (`calculate_average`, `grades_list`, `avg_score`), которые отражают их назначение.
    -   После объявления функции (`def`) и перед началом ее тела оставляй две пустые строки. Между логическими блоками внутри функции оставляй одну пустую строку для улучшения читаемости.
