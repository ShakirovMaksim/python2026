# Практическое задание 35.3

Это задание относится к третьему блоку урока, посвященному `*args`, `**kwargs` и распаковке. В предыдущих блоках вы научились работать с позиционными и именованными аргументами, а также с безопасными значениями по умолчанию. Сейчас вы переходите к созданию гибких функций, способных обрабатывать переменное количество входных данных — ключевой навык для написания универсального кода.

## Задания

### Описание задачи
Вы разрабатываете модуль для обработки данных о студентах. Данные могут поступать в разном формате: иногда это просто имя и оценка, иногда — имя, оценка и дополнительные теги (например, "отличник", "актив"), а иногда — имя, оценка и произвольные метаданные в виде пар ключ-значение (например, `city="Казань"`, `attendance=0.95`).

Ваша задача — написать функцию `process_student`, которая:
1.  Обязательно принимает имя студента (`name`) и его оценку (`grade`).
2.  Может принимать произвольное количество дополнительных тегов через позиционные аргументы. Эти теги должны собираться в кортеж.
3.  Может принимать произвольное количество дополнительных метаданных через именованные аргументы. Эти данные должны собираться в словарь.
4.  Функция должна возвращать словарь, структурированный следующим образом:
    ```python
    {
        "name": <имя>,
        "grade": <оценка>,
        "tags": <кортеж тегов или пустой кортеж, если тегов нет>,
        "meta": <словарь метаданных или пустой словарь, если метаданных нет>
    }
    ```

Также вам нужно написать функцию `print_student_summary`, которая:
1.  Принимает словарь студента (результат работы `process_student`).
2.  Распаковывает этот словарь, чтобы передать его данные в форматированную строку для печати. Используйте распаковку словаря (`**`).
3.  Печатает информацию в формате: `Студент: {name}, Оценка: {grade}, Теги: {tags}, Дополнительно: {meta}`.

### Пример входных и выходных данных

**Пример 1: Базовый вызов**
```python
# Входные данные (вызов функции)
student1 = process_student("Анна", 5)
print_student_summary(student1)

# Ожидаемый вывод в консоли
Студент: Анна, Оценка: 5, Теги: (), Дополнительно: {}
```

**Пример 2: С тегами и метаданными**
```python
# Входные данные (вызов функции)
student2 = process_student("Иван", 4, "актив", "олимпиадник", city="Москва", year=2)
print_student_summary(student2)

# Ожидаемый вывод в консоли
Студент: Иван, Оценка: 4, Теги: ('актив', 'олимпиадник'), Дополнительно: {'city': 'Москва', 'year': 2}
```

**Пример 3: Использование распаковки при вызове**
```python
# Входные данные
tags_list = ["математик", "исследователь"]
meta_dict = {"стипендия": True, "факультет": "ФИИТ"}
student3 = process_student("Мария", 5, *tags_list, **meta_dict)
print_student_summary(student3)

# Ожидаемый вывод в консоли
Студент: Мария, Оценка: 5, Теги: ('математик', 'исследователь'), Дополнительно: {'стипендия': True, 'факультет': 'ФИИТ'}
```

### Критерии проверки и ограничения
1.  Функция `process_student` должна корректно использовать `*args` для сбора тегов и `**kwargs` для сбора метаданных.
2.  Функция `print_student_summary` должна использовать распаковку словаря (`**`) для передачи аргументов в `print` или f-строку.
3.  Если теги или метаданные не переданы, в итоговом словаре должны быть пустой кортеж и пустой словарь соответственно.
4.  **Важное ограничение:** В решении нельзя использовать темы из будущих блоков, такие как `global`, замыкания или обсуждение области видимости (кроме базового понимания, что переменные внутри функции — локальные). Фокус строго на `*args`, `**kwargs` и распаковке.

### Решение задачи (псевдокод)

**Для функции `process_student`:**
1.  Объявить функцию с параметрами: `name`, `grade`, `*tags`, `**meta`.
2.  Создать пустой словарь `result`.
3.  Записать в `result` по ключу `"name"` значение параметра `name`.
4.  Записать в `result` по ключу `"grade"` значение параметра `grade`.
5.  Записать в `result` по ключу `"tags"` кортеж, полученный из `tags`.
6.  Записать в `result` по ключу `"meta"` словарь, полученный из `meta`.
7.  Вернуть словарь `result`.

**Для функции `print_student_summary`:**
1.  Объявить функцию с параметром `student_dict`.
2.  Используя распаковку словаря, получить значения по ключам `name`, `grade`, `tags`, `meta` из `student_dict`.
3.  Напечатать строку, подставив полученные значения в заданный формат.

### Рекомендации
-   Вынести в отдельную функцию: Задание уже структурировано вокруг двух функций. Убедитесь, что логика каждой функции четко ограничена своей задачей: `process_student` — сбор и структурирование данных, `print_student_summary` — форматированный вывод.
-   PEP8:
    1.  Используйте осмысленные имена параметров: `*tags` и `**meta` ясно указывают на их содержимое.
    2.  После определения функции оставляйте две пустые строки перед следующим блоком кода (например, перед вызовами функций для тестирования).
