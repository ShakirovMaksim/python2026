# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Коворкинг ведёт список переговорок и время их последнего бронирования. Администратор хочет быстро подсветить занятые и свободные комнаты. Создай словарь last_booking, где ключ — название переговорки, а значение — дату и время последнего бронирования. Затем выведи информацию для переменных room_to_check и new_room.

## Пример входных данных
```python
rooms = [
    ('Omega', '2025-12-05 14:00'),
    ('Delta', '2025-12-05 16:30'),
    ('Atlas', '2025-12-04 11:15'),
]
room_to_check = 'Delta'
new_room = 'Nova'
```

## Пример выходных данных
```text
Delta -> 2025-12-05 16:30
Nova -> еще свободна
```

## Критерии успешного выполнения
- Список пар преобразован в словарь last_booking с ключом 'название переговорки'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для room_to_check и new_room в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку rooms в цикле и наполняй словарь last_booking.
- Используй множественное присваивание: key, value = запись из rooms.
- Для отсутствующих данных возвращай строку 'еще свободна'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
