# Задание 33.1. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Введение в генераторы коллекций» помогает увидеть. где компактное выражение избавляет от вспомогательных переменных и временных списков.

## Условие задачи
**Сложность**: <span style="color: #c62828">● сложный</span>. примерное время: 15–20 минут

Получен список заказов интернет-магазина в формате `[(client, amount, city), ...]`. Нужно сформировать список текстов `"client купил на amount₽"` только для заказов из городов. чьи названия начинаются на букву `К`. Суммы больше 50 000 обозначь как `amountK` (например. `52000K`). Реализуй всё одной конструкцией list comprehension.

## Пример входных данных
```
orders = [
    ("Николай", 32000, "Казань"),
    ("Анна", 52000, "Самара"),
    ("Ирина", 61000, "Калуга"),
    ("Марк", 15000, "Кемерово")
]
```

## Пример выходных данных
```
['Николай купил на 32000₽', 'Ирина купила на 61000K', 'Марк купил на 15000₽']
```

## Критерии успешного выполнения
- Использован один list comprehension без дополнительных циклов.
- Отбираются только города. начинающиеся на `К` или `к`.
- Суммы выше 50 000 получают суффикс `K`. остальные выводятся как есть.
- Пол сохраняет род в тексте: для имён на `а` или `я` используй `купила`. иначе `купил`.

## Подсказки
- Можно заранее определить функцию или выражение для выбора текста `купил/купила` и вызвать её внутри comprehension.
- Проверь город с помощью `city.lower().startswith("к")`.

## Частые ошибки и ограничения
- Не добавляй условие `if` в двух местах. достаточно одного фильтра.
- Не модифицируй исходный список `orders`.
- Не используй `append`. результат должен быть создан comprehension-ом.
